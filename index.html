1. Внешний сервис присылает JSON или HTTP код, где описаны параметры уведомления и его содержание.
  2. Сервис уведомлений занимается отправлением уведомлений с учетом настроек пользователей. Cохраняет отправленные сообщения в базы данных и ведет логи активности.
  3. Сервис настроек пользователя представляет собой либо сторонний сервис, либо UI, где пользователь может подписаться на уведомления, запланировать их график и т.д. Соответствующие настройки сохраняются в базу данных.
  4. Очередь сообщений хранит в себе все уведомления для отправки.
  5. Worker - это простые функции, основная задача которых - отправка уведомлений в соответствующий сервис. Также передают статус отправления уведомления в базу данных.
  6. Сторонние сервисы, обрабатывающие конкретный тип уведомлений и отправляющие их конечному пользователю.
  7. Служба, проверяющая статус отправки уведомлений. Возвращает уведомления в очередь при необходимости.

    Плюсы:

  • Масштабируемость (за счет Worker);
  • Надежность (за счет очереди сообщений и переотправки неотправленных уведомлений).

    Минусы:

  • Отсутствует регулировка входящих запросов (сервис уведомлений может быть перегружен);
  • Нет возможности сортировать уведомления по приоритету, а также разделять уведомления на массовые и одиночные;
  • База данных может стать очень объемной.
